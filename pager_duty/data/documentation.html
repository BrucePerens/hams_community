1. Architecture Overview (CQRS)

The Pager Duty module operates on a Command Query Responsibility Segregation (CQRS) model. Odoo acts as the control plane to manage configurations, while an isolated, standalone Python daemon (generalized_monitor.py) executes the actual polling and health checks outside of Odoo's web workers.

2. Configuration Wizard

Administrators can configure all monitoring endpoints graphically. Navigate to Ham Admin > System Health > Monitoring Checks. Click Edit YAML Config File to synchronize the database records with the daemon's physical pager_config.yaml file.

3. Supported Check Types

  • System Resource: Monitors Disk, Memory, CPU, IO Wait, and Hypervisor Steal percent limits.
  • DNS Resolution: Performs un-cached, full-chain root server lookups (via dig +trace).
  • HTTP / HTTP/3 (QUIC): Validates web server responsiveness across modern and legacy web protocols.
  • TCP & UDP Sockets: Asserts raw datagram and stream connectivity, supporting hexadecimal payload injection.
  • Native Protocol Handshakes: Built-in support for executing strict protocol validations against Redis, RabbitMQ (AMQP), XML-RPC, and JSON-RPC.
  • PostgreSQL DB: Validates database connection pools directly via psycopg2.
  • SSL/TLS Certificate Expiry: Alerts administrators if the domain's certificate expires within a critical threshold.
  • Anomaly Detection: Evaluates a custom SQL query. If the integer drops below the threshold, it fires an alert.
  • Synthetic Journeys: Executes a custom shell script or headless browser test, alerting on non-zero exit codes.
  • Pre-Flight & Dry-Runs: Safely simulates critical infrastructure maintenance including certbot --dry-run, pg_dump -s, nginx -t, and outbound SMTP authentications.
  • ICMP Ping: Basic network reachability testing for remote gateways and routers.
  • Docker Container Health: Monitors the native execution state of background Docker containers.
  • Systemd Services: Validates that critical host OS services remain active.
  • Memcached Server: Raw TCP socket handshakes to ensure the cache is responding to statistic inquiries.
  • SSH Handshake: Validates raw port accessibility and protocol compliance for remote access.

4. Intelligent Calendar Routing

The module natively ties into the Odoo Calendar. By checking the "Is Pager Duty Shift" box on an event, the system automatically knows exactly who is on-call at any given minute. It fully supports complex recurrence rules (like "the first Tuesday of the month") and manages shift-handoffs seamlessly without requiring manual configuration files.

5. Fail-Safes & Self-Healing

Checks can be configured with a Startup Grace Period (e.g., 120 seconds) to prevent false-positive alerts while Odoo is initially booting. If Odoo crashes entirely, the daemon utilizes an Airgapped SMTP Fallback and Discord/Slack Webhooks to bypass the ERP and alert the administrator directly.

Furthermore, the daemon features a self-healing dependency scanner. For example, if it is instructed to test a Zero Trust tunnel but the cloudflared binary is missing from the system, it will automatically download the static binary from GitHub and continue execution without requiring manual intervention.

6. Advanced SRE Capabilities

  • Cascading Failure Suppression: Link checks together via "Parent Check" dependencies. If the master database goes offline, downstream HTTP checks will be automatically silenced to prevent alarm fatigue.
  • Maintenance Windows: Schedule explicit silences via Maintenance Start/End dates to mute alerts during planned infrastructure work.
  • Push Monitoring (Heartbeat): Use the /api/v1/pager/heartbeat/<uuid> endpoint to actively listen for successful "pings" from external backup scripts. If a ping is missed, the daemon raises an incident.
  • Multi-Tier Escalation: A background cron job constantly evaluates open incidents. If a critical incident remains unacknowledged for over 15 minutes, it automatically escalates the alert directly to the entire administrative group, bypassing the scheduled shift.
  • Analytics: Incident records automatically track Mean Time To Acknowledge (MTTA) and Mean Time To Resolve (MTTR) metrics for operational review.
