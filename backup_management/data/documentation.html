Unified Backup Management Manual

This module implements the Best-in-Class Hybrid Architecture for system and database backups. It orchestrates Kopia for file/system backups and pgBackRest for PostgreSQL continuous WAL archiving.

1. Self-Healing Deployment

Odoo expects the kopia and pgbackrest packages to be installed at the host OS layer. If an administrator fails to install Kopia during provisioning, the application will automatically perform a Just-In-Time (JIT) download of the static Kopia binary from GitHub, placing it safely into the Odoo datastore to guarantee operation.

2. Configurations & Orchestration

Navigate to Ham Admin > Backups > Configurations. You can define targets, storage backend credentials (S3/B2), and retention policies for both Kopia and pgBackRest. If using Kopia, enter your repository password here; it will be symmetrically encrypted at rest.

You may also actively trigger backups or push new retention policies directly to the underlying storage engines via the action buttons in the configuration header.

2. Synchronization

An automated hourly cron job interrogates the underlying CLI tools (kopia snapshot list and pgbackrest info) to fetch the latest backup metadata and populate the Snapshots table.

3. Restoration Assistance

Because restoration is highly destructive, it must be performed by a DBA via the CLI. Navigate to any Snapshot record to view and copy the exact generated restoration command for that specific point in time.

4. Pager Duty Synergy & Stale Backups

This module features native synergy with the Pager Duty SRE suite. If an underlying backup engine fails to execute, or if the system detects that a configuration hasn't produced a new successful snapshot in over 26 hours, it will automatically open a Critical incident, ensuring the on-call administrator is immediately paged.

5. Snapshot Size Anomaly Detection

Administrators can configure a Minimum Expected Size (MB) on each backup configuration. When a new snapshot is ingested, the system compares its physical byte size against this threshold. If the snapshot is anomalously small (indicating a missing mount or failed filesystem read), the system immediately fires an SRE alert to prevent a false sense of security.
