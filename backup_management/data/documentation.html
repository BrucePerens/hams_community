Backup Management User Guide

Introduction

The Backup Management module allows you to configure, monitor, and manage automated backups for your system. It supports both file-level backups (Kopia) and database backups (pgBackRest).

Backup Configurations

A Backup Configuration defines where and how your data is backed up. To create one, go to Backup Management > Configurations.

  • Engine: Choose between Kopia (files) or pgBackRest (database).
  • Target / Stanza: The path to your backup repository or the name of your pgBackRest stanza.
  • Website: Link the configuration to a specific website for multi-tenant isolation.
  • Storage Type: (Kopia only) Choose Local, AWS S3, or Backblaze B2.
  • Retention Policies: Set how many daily, weekly, and monthly backups to keep.

Monitoring Backups

The Backup Dashboard provides a real-time overview of all backup configurations. It shows the latest snapshot, its size, and status. If a backup is older than 26 hours, it will be marked as stale.

Manual Operations

While backups are automated via scheduled tasks, you can manually trigger operations from the configuration form:

  • Trigger Backup: Starts a new backup immediately.
  • Sync Snapshots: Refreshes the list of snapshots from the backup repository. [@ANCHOR: UX_BACKUP_SYNC]
  • Apply Policies: Forces the retention policies to be applied (deleting old snapshots).

Restoring Data

To restore data, navigate to the Snapshots tab of a configuration, select a snapshot, and click the Restore button. You will be asked to provide a target path where the data should be restored.

Security & Safety

The system prevents access to sensitive system directories (like /etc or /root) to ensure that backup operations cannot compromise the server. All passwords and keys are encrypted.
