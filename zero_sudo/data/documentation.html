Zero-Sudo Security Core

The zero_sudo module is the foundational security layer for the entire ecosystem. It enforces the strict Zero-Sudo Architecture (ADR-0002) and Service Account Web Isolation (ADR-0005) to ensure least-privilege execution across all boundaries.

Centralized Document Installation Facility

This module provides a generalized document installation facility to automatically provision technical and user manuals into the manual_library or knowledge systems when the Odoo registry loads.

Implementation & Usage

Modules no longer require ad-hoc post_init_hook scripts to install documentation. Instead, developers simply need to declare a hard dependency on zero_sudo and add the knowledge_docs array to their __manifest__.py:


"knowledge_docs": [
    {
        "name": "Your Module Guide",
        "path": "your_module/data/documentation.html",
        "icon": "📄",
        "category": "workspace"
    }
]

The facility runs automatically on registry load. It calculates a SHA-256 hash of your documentation file and securely stores it via Zero-Sudo system parameters to guarantee idempotency. If you update the HTML file in the future, the hash mismatch will automatically trigger a clean update of the live article.

If neither the manual_library nor knowledge modules are installed, the injection sequence is safely bypassed.
