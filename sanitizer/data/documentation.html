Ham Sanitizer Override Documentation

This module overrides the default Odoo HTML sanitization behavior to allow <script>, <iframe>, and <dfn> tags.

Key Features

  • Expanded Tag Set: Adds <script>, <iframe>, and <dfn> to the whitelist of safe HTML tags.
  • Rich Attribute Support: Allows critical attributes for third-party embeds, including src, allow, allowfullscreen, async, and defer.
  • Bootstrap Integration: Whitelists data-bs-toggle to support interactive UI elements defined directly in HTML content.
  • Low-Level Protection Bypass: Monkey-patches the underlying lxml cleaner to ensure scripts and frames are preserved even during deep sanitization passes.

How to Use

  1. Installation: Install the module via the Odoo Apps menu. No further configuration is required.
  2. Content Creation: Navigate to any HTML field (e.g., a Website Snippet, Blog Post, or Forum Thread).
  3. Inserting Dynamic Content: Switch to the HTML/Code view and paste your script or iframe. For example: 
    <iframe src="https://www.youtube.com/embed/..." allowfullscreen></iframe>
  4. Saving: Upon saving, Odoo will no longer strip these tags, allowing them to render on the frontend.

Multi-Tenant Awareness

Note: This is a Global Module. The configuration changes applied here affect all companies and websites in this Odoo instance. It is not possible to enable these tags for one website while keeping them disabled for another.

Critical Security Warning

This module intentionally disables Odoo's primary defense against Cross-Site Scripting (XSS). By allowing <script> tags, any user with HTML editing privileges can execute arbitrary JavaScript in the browsers of other users, including administrators.

Deployment Checklist:

  • Is the environment private?
  • Do you trust 100% of the users with "Editor" or "Designer" access?
  • Are you prepared for the risk of session hijacking if a user account is compromised?
