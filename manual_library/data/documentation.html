Manual Library Documentation

Version: Odoo 19 Community
Summary: A clean-room, open-source implementation of a hierarchical documentation and knowledge-base system for Odoo Community.

1. System Overview

The Manual Library provides a drop-in replacement for the proprietary Odoo Enterprise knowledge module. It allows other modules to install and manage documentation via standard Odoo XML data loading without requiring an Enterprise license.

  • Hierarchical Documentation: Organize articles in a nested tree structure for easy navigation.
  • API Interoperability: Implements the core knowledge.article API so XML data files from other modules install seamlessly.
  • Rich Text Editing: Utilizes Odoo's standard HTML editor for content creation.

2. Security & Access Control

The module implements a granular three-tier access system:

  1. Manual Administrator: Full CRUD (Create, Read, Update, Delete) access to all articles and hierarchies.
  2. Internal User (Standard): Contextual read access to articles marked as 'workspace' or 'shared' based on internal permissions. Cannot mutate data.
  3. Public Guest: Contextual read access exclusively to articles where is_published is True. Strictly blocked from all mutation attempts.

3. Regulatory Compliance Statement

How the manual_library module complies with regulations:

  • GDPR (Data Minimization): The module operates purely on internal documentation and public guides. The frontend feedback system tracks "Helpful/Unhelpful" votes anonymously without logging Personally Identifiable Information (PII) or IP addresses.
  • WCAG 2.1 AA (Accessibility): The dynamically generated frontend Table of Contents (TOC) and hierarchical sidebar use proper semantic HTML nesting (<nav>, <ul>, <li>). Interactive search forms and dropdown buttons include explicit aria-label attributes for screen readers.
  • Clean-Room Implementation (Legal Compliance): The module legally complies with the Abstraction-Filtration-Comparison test to provide API interoperability. It does not contain any literal or non-literal copied code, logic, or proprietary UI designs from Odoo Enterprise.
  • Frontend Framework Compliance: Adheres to modern Odoo 19 standards by strictly utilizing Vanilla JS and standard OWL concepts for the TOC generation, completely avoiding deprecated jQuery dependencies.
